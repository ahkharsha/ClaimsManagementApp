Project Objective
The objective of this project is to develop a smart web-based platform that enables insurance 
companies to efficiently manage policy issuance, claims processing, customer onboarding, 
agent performance, and regulatory compliance using modern full-stack technologies, AI/ML 
fraud detection, and real-time alerts.
The system aims to: • Automate end-to-end policy lifecycle management — from quote to renewal • Streamline claims intake, investigation, approval, and settlement workflows
• Manage customer profiles, agent assignments, and policy portfolios
• Detect fraudulent claims using Python-based Machine Learning models
• Provide real-time notifications for claims status updates, renewals, and alerts
• Generate interactive analytics dashboards for claims trends and agent performance
• Centralise all insurance operations — policies, claims, agents, and customers — in one 
platform
This project demonstrates the integration of: • Frontend development using ReactJS
• Enterprise backend APIs using Spring Boot • Core Java concepts like multithreading and bulk file processing
• Python AI/ML fraud detection and risk scoring via FastAPI • Real-time communication using NodeJS and Socket.IO
• Cloud deployment using AWS (EC2 + S3) • Data ingestion and analytics processing via CSV/ETL pipelines
The outcome is a scalable, cloud-ready insurance intelligence system capable of improving 
claims cycle time, fraud prevention, agent productivity, and customer satisfaction through datadriven automation.
System Overview
Develop an intelligent insurance operations platform that helps: • Admins manage products, agents, and company-wide policy portfolios
• Agents onboard customers, issue policies, and track their sales pipelines
• Customers view their policies, file claims, and track settlement status
• AI detects fraudulent claims and scores risk for underwriting decisions
• Real-time notifications keep all stakeholders updated on critical events

2. Policy & Customer Management
Admin Can
• Create and manage insurance product types (Health, Motor, Life, Property) • Define coverage rules, premium slabs, and policy terms
• Assign agents to regions and customer segments
• Upload policy and customer data via CSV bulk import • View platform-wide policy portfolio and renewal calendar
Agent Can
• Onboard new customers and collect KYC documents
• Generate quotes and issue policies for assigned customers
• Track policy renewals, lapses, and due premiums
• Add endorsements and policy amendments
• View personal sales performance and commission statements
Customer Can
• View all active and lapsed policies in personal dashboard
• Download policy documents and premium receipts
• Request policy renewals and endorsements
• File new claims and track existing claim status
• View agent contact details and appointment history
3. Claims Processing & Workflow
Claims Lifecycle
Stage Actor Action
1 — Intake Customer / Agent Submit claim with supporting documents
2 — Registration System Auto-assign claim ID and notify surveyor
3 — Survey Surveyor / Agent Inspect, validate, and upload survey report
4 — Assessment Claims Manager Review survey, run fraud score, approve/reject
5 — Settlement Finance Process payment and close claim
6 — Closure System Update records and notify customer
Claims Manager Can
• View all pending, in-review, and settled claims
• Assign surveyors to field inspection tasks
• Review AI fraud detection score before approval • Approve, reject, or escalate claims with comments
• Generate settlement letters and payment instructions
Key Claim Types Supported
• Motor Accident Claims
• Health / Hospitalisation Claims
• Property Damage Claims
• Life Insurance Death Claims

Sample Feature Set for ML Model
Feature Description
claim_amount Claimed amount in INR
days_since_policy_start Number of days between policy start and claim date
previous_claims_count Total prior claims by the same customer
claim_type Motor / Health / Property / Life (encoded)
customer_age Age of policyholder at time of claim
policy_premium_ratio Claim amount divided by annual premium paid
surveyor_mismatch_flag 1 if survey report contradicts claim (rule-based)
